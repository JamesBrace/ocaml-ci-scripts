!?? quip command_pattern is a function of command_pattern parameters which behaves as quip but does not terminate if quip command_pattern ... returns a non-zero exit code. The exit code can be retrieved with {?$} "?".

val (!?*) : (string ‑> 'b, 'b) exec ‑> ('a, 'b * int) exec

!?* quip command_pattern is a function of command_pattern parameters which behaves as quip but does not terminate if quip command_pattern ... returns a non-zero exit code. The exit code is the second element of the returned pair.

val (?$) : string ‑> string

?$ ENV_VAR looks up environment variable ENV_VAR like getenv_default but terminates execution with an error if ENV_VAR is not found.

  • ?$ "@" is this program's arguments.
  • ?$ "?" is the last subprocess's exit code.
